# Agent Guidelines

## Critical Requirements

- **Always use Emerald Angular Components from the selected design system.** When implementing components, prioritize using the V2 versions where available (e.g., use `emerald-date-picker-v2` instead of `emerald-date-picker`).
- Use design tokens from the design system `CBRE Emerald Angular Design System` rather than creating new css properties.
- Always verify the code compiles with `npm run build` as a final step after making a code change.

## Design System Reference

For detailed information about available components and their usage, refer to the `CBRE Emerald Angular Design System` file which provides comprehensive documentation on:

- Component architecture
- All available components and their purposes
- Integration patterns
- Accessibility considerations
- Responsive design guidelines

Always consult the corresponding MDX documentation files in the `design-system-v2-docs` folder for specific implementation details.
